npm 依赖包版本锁定

背景

项目遇到第三方依赖升级到 babel@v7.20，导致项目中的 generator 转义存在问题。

解决方案

同理某些依赖包升级与当前系统脚手架不兼容的情况，使用 npm@v8.3+ 支持的 overrides 来锁定包版本，这样所有依赖树中对应的包都会限制为指定版本。

以 babel 举例：

移除原来的 node_modules 和 package-lock.json ，使用 npm i 重新安装后，查看 package-lock.json 可以发现相关包版本已被限制为指定版本。

总之，还是 pnpm 大法好，一次性解决此类问题，速度也有较大提升。
